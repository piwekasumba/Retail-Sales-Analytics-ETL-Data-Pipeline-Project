# Retail Sales Data Pipeline (PostgreSQL)

## Project Overview

This project demonstrates a PostgreSQL-based data pipeline that transforms raw retail sales data into structured, analysis-ready datasets.

It follows a simplified ETL workflow (Extract → Transform → Load), focusing on data ingestion, cleaning, relational structuring, and analytical reporting using SQL.

The emphasis is on data preparation and transformation logic rather than only querying, simulating a foundational data engineering workflow.

---

## Key Focus Areas

- Loading raw CSV data into PostgreSQL staging tables
- Cleaning and standardizing inconsistent or missing data
- Designing structured relational tables for analysis
- Applying SQL transformations to prepare datasets for reporting
- Building aggregated datasets for business insights

---

## Skills Demonstrated

SQL • PostgreSQL • ETL Concepts • Data Cleaning • Data Modeling • Data Transformation • Analytical Querying • Reporting

---

## Project Highlights

- Transformed raw retail sales CSV data into structured PostgreSQL tables
- Built SQL-based transformation logic for cleaning and standardisation
- Created analytical query layers using aggregation techniques
- Analysed sales performance across products, categories, regions, and time periods
- Generated structured outputs for reporting and decision support

---

## Analytical Outputs

- Total sales by product, category, and region
- Monthly and yearly sales trend analysis using SQL date functions
- Top-performing products based on revenue and quantity sold
- Customer purchase summaries using aggregation queries
- Regional sales distribution analysis using GROUP BY

---

## Project Structure

This project follows a simplified ETL workflow:

- Raw Layer: raw CSV data loaded into staging table
- Staging Layer: data cleaning and standardisation using SQL
- Transformation Layer: structured SQL queries for analytical preparation
- Output Layer: aggregated reporting queries for insights

## Conclusion

This project demonstrates foundational SQL and ETL skills used in data analytics workflows, focusing on transforming raw data into structured datasets suitable for reporting and business analysis.
